# 第一期开箱预设与器械图标设计

## 目标

将用户确认的器械与动作写入应用默认预设，并使用本地、统一线性风格的 SVG 展示第一期器械图标。

## 默认预设

新增或补全以下器械与动作：

- 蝴蝶机：夹胸、反向飞鸟。
- 徒手：引体、悬垂举腿。
- 杠铃：牧师凳弯举。
- 分夹腿训练器：髋内收、髋外展。
- 高位下拉机：宽距高位下拉、窄距高位下拉、颈后下拉。
- 龙门架：绳索划船、三头下压、绳索夹胸、绳索弯举。
- 划船机：划船。

预设动作均采用 `weight-reps` 模板，徒手动作和划船机记录也保留该模板，以便用户按需输入重量或阻力。

## 图标

- 为上述七种器械创建本地 SVG 图标，采用同一线宽、圆角端点与单色描边。
- 图标放在 `src/assets/icons/`，文件名使用 `equipment-<slug>.svg`。
- `Equipment.icon` 保存图标的 slug；首页用静态映射解析图标资源。
- 对未配置图标的旧器械，首页继续显示通用“🏋️”占位，保证已有自定义数据兼容。

## 数据初始化与兼容性

- 默认数据初始化不再只在表为空时整批写入；改为按 id 补充缺失的器械和动作，因此升级后的现有数据库可获得第一期预设，不会覆盖、修改或删除用户已有数据。
- 已存在的同 id 记录保持原样。
- 为已有器械（徒手、杠铃、龙门架）在初始化时补充图标 slug，不影响名称和动作。

## 验证

- 扩展模板 Node 测试，确认新增预设 id 与图标映射均可解析。
- 运行 `node --test tests/*.test.mjs` 与 `npm run build`。
- 手动检查首页七种器械的图标显示和器械抽屉中的预设动作。
