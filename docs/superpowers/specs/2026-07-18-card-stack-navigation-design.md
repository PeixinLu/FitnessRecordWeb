# 一级页面卡片堆叠导航设计

## 目标

移除记录、日历、统计、设置四个一级页面的底部导航栏，将它们作为固定卡片常驻在同一个页面容器中。用户通过右下角悬浮按钮进入或退出 3D 堆叠态，并在堆叠态点击目标卡片，将该页面稳定召至前台。

二级管理流程继续使用现有 Vue Router，包括器械管理、动作管理、训练详情独立路由和数字滚轮调试页。一级页面切换不写入路由，也不提供浏览器前进、后退或深链能力。

## 范围

### 包含

- 删除现有悬浮底部导航栏及其底部空间。
- 在单个一级页面容器内常驻 `Home`、`Calendar`、`Statistics`、`Settings`。
- 实现前台态、堆叠态、悬浮切换按钮和卡片点击召回。
- 在堆叠态统一屏蔽页面内部的指针、键盘与焦点交互。
- 保留一级页面内部状态，例如日历选择、器械分页和滚动位置。
- 保留二级路由及其现有进入方式。
- 支持系统减少动态效果偏好。

### 不包含

- 不实现参考 Demo 的动态添加或删除页面。
- 不将器械管理、动作管理、训练编辑等二级流程加入卡片堆栈。
- 不保存或恢复刷新前的一级页面索引；页面加载默认展示记录页。
- 不增加左右滑动切页或拖拽手势。

## 方案选择

采用“一级页面内部状态 + 二级页面保留路由”的混合架构。

四个一级页面固定挂载，由 `activeIndex` 和 `isStacked` 两个状态决定位置与交互能力。路由只在进入二级页面时使用。相比为每张卡片同步路由，该方案与参考 Demo 的状态模型一致，能保留页面局部状态，并减少动画与路由生命周期之间的耦合。

## 页面与路由架构

`App.vue` 根据当前路由是否属于一级入口决定渲染内容：

- `/` 渲染 `PrimaryPageStack`，承载全部四个一级页面。
- 历史一级地址 `/calendar`、`/statistics`、`/settings` 重定向到 `/`，避免旧书签进入空白页。
- `/equipment-management`、`/equipment-management/:equipmentId`、`/workout/:date/:exerciseId`、`/debug/number-wheel` 继续通过 `router-view` 渲染。

一级页面内部需要打开二级流程时，继续调用现有 Router API。用户从二级路由返回 `/` 后，一级页面容器重新挂载并默认展示记录页；本次改造不额外实现跨二级路由的一级状态恢复。

## 组件边界

### `PrimaryPageStack.vue`

职责：

- 定义四个固定一级页面的顺序和可访问名称。
- 维护 `activeIndex` 与 `isStacked`。
- 计算每张卡片的 transform、层级和外观状态。
- 承载卡片级召回层与 FAB。
- 统一控制交互隔离和无障碍属性。

该组件不读取或修改训练业务数据；业务逻辑仍留在各自页面中。

### `cardStack.ts`

职责：

- 以纯函数计算每张卡片在前台态或堆叠态下的视觉状态。
- 输出 transform、圆角、阴影、层级、是否激活以及是否允许页面内容交互。
- 隔离公式和状态判断，便于通过 Node 测试覆盖。

### 四个现有一级页面

保持业务职责不变。页面不需要自行监听堆叠状态，也不需要为每个按钮增加禁用判断。必要的尺寸调整仅用于让页面填满卡片容器并由页面内部管理滚动。

## 状态模型

```ts
interface CardStackState {
  activeIndex: number
  isStacked: boolean
}
```

- 初始状态：`activeIndex = 0`、`isStacked = false`。
- FAB 在前台态和堆叠态之间切换，不修改 `activeIndex`。
- 堆叠态点击第 `i` 张卡片：设置 `activeIndex = i`，随后设置 `isStacked = false`。
- 前台态的卡片级召回层不可交互，只有激活页面内容可以交互。

## 视觉与动画

Stage 遵循参考文档：

- 宽高占满视口，优先使用 `100dvh` 并以 `100vh` 兜底。
- 背景为 `#000`。
- `perspective: 80px`。
- `overflow: hidden`。

卡片遵循参考文档：

- 绝对定位并占满 Stage。
- `transform-origin: top`。
- `transform-style: preserve-3d`。
- 堆叠态圆角 `20px`，阴影 `0 30px 80px rgba(0, 0, 0, 0.35)`，旋转 `rotateX(-1deg)`。
- 四卡堆叠时 Y 偏移依次为 `10px`、`25%`、`50%`、`75%`。
- Z 深度依次为 `-12px`、`-11px`、`-10px`、`-9px`。
- 前台激活页位于 `translate3d(0, 0, 0)`，无圆角和阴影。
- 前台非激活页按其与激活页的相对位置滑到视口上方或下方。
- 前台到堆叠使用 400ms `cubic-bezier(0.25, 0.8, 0.25, 1.08)`。
- 堆叠到前台使用 300ms `cubic-bezier(0, 0, 0.2, 1)`。
- transition 不包含 `z-index`。

FAB 固定在右下角，尺寸为 56px，层级高于卡片。位置同时考虑 `env(safe-area-inset-right)` 与 `env(safe-area-inset-bottom)`。图标在进入堆叠态后反向，表达恢复前台的动作。

当 `prefers-reduced-motion: reduce` 生效时，卡片和图标过渡缩短为近乎即时，不播放明显的 3D 移动效果。

## 堆叠态交互隔离

堆叠态必须满足“页面内容零交互，卡片召回层唯一可交互”的不变量。

每张卡片分为两层：

1. 页面内容层：承载现有 Vue 页面组件。
2. 卡片召回层：覆盖整张卡片的透明按钮，仅在堆叠态启用。

堆叠态下页面内容层同时应用：

- `pointer-events: none`，屏蔽点击、触摸和页面手势。
- `inert`，屏蔽键盘焦点与辅助技术交互。
- `aria-hidden="true"`，避免屏幕阅读器读取多张页面内容。

召回层阻止事件冒泡，点击后只选择对应卡片并退出堆叠态。召回层提供“打开记录页面”“打开日历页面”等可访问名称，并支持原生按钮的 Enter 与 Space 键行为。

前台态下：

- 激活页面内容层移除 `inert` 和 `aria-hidden`，恢复 `pointer-events`。
- 其他页面继续保持 `inert` 和 `aria-hidden`，即使位于视口外也无法获得焦点。
- 所有卡片召回层关闭指针事件并退出 Tab 顺序。

Vant 抽屉和弹窗通过 Teleport 渲染在 Stage 外，其现有遮罩层级高于 FAB。用户需要先关闭编辑流程，才能点击 FAB 进入堆叠态。

## 数据流

1. `App.vue` 在根路由渲染 `PrimaryPageStack`。
2. `PrimaryPageStack` 根据本地状态为每张卡片计算样式和交互属性。
3. FAB 只翻转 `isStacked`。
4. 堆叠召回层选择卡片时更新 `activeIndex`，再退出堆叠态。
5. 页面业务组件继续直接使用现有 Pinia Store；切换一级页面不会重新加载 Store 或卸载组件。
6. 页面进入二级路由时由现有 Router 接管渲染。

## 异常与边界处理

- 卡片索引计算函数对越界的激活索引进行钳制，防止未来调整页面数组时出现无前台页面。
- 页面数量固定且非空；若未来扩展为空数组，应由容器拒绝渲染 FAB，而不是执行除零位置计算。
- 快速连续点击 FAB 时，视觉状态始终从响应式状态重新计算，不维护额外动画队列。
- 快速连续点击召回层时，第一次点击退出堆叠后召回层立即失效，后续点击不会落入页面业务内容。

## 测试策略

### 自动测试

使用现有 `node:test` 风格新增测试：

- 四卡堆叠位置与 Z 深度符合规范。
- 前台激活卡片全屏，其他卡片按上下方向移出视口。
- 堆叠态所有页面内容均不可交互，只有召回层启用。
- 前台态只有激活页面内容可交互。
- FAB 切换不改变激活索引，卡片召回同时更新索引并退出堆叠态。
- `App.vue` 不再包含底栏结构与底部预留空间。
- 根容器包含四个固定一级页面和交互隔离属性。
- 历史一级路由重定向到 `/`，二级路由保持存在。

### 构建验证

运行：

```bash
node --test tests/*.test.mjs
npm run build
```

### 手动验证

在移动端视口验证：

1. 首次加载显示记录页，底部导航栏消失。
2. 点击 FAB 后四张卡片按规范进入堆叠态。
3. 在堆叠态点击页面内部按钮所在位置，不执行原业务，只召回所点卡片。
4. 分别召回记录、日历、统计、设置页面，动画方向和层级正确。
5. 切换前后各页面滚动位置和局部选择状态保留。
6. 设置页仍能进入器械管理，二级页面返回后回到根容器。
7. Vant 抽屉或弹窗打开时无法误触 FAB。
8. iPhone SE 高度和带安全区设备上，FAB 不遮挡系统区域。
9. 开启减少动态效果后不出现明显 3D 位移动画。

## 验收标准

- 一级页面底部导航栏及其占位完全移除。
- 四个一级页面由单一容器常驻管理，不依赖一级路由切换。
- 前台态与堆叠态的尺寸、位置、层级和过渡符合参考规范。
- 堆叠态下无法触发任何页面内部业务交互。
- 点击任意可见卡片区域只会将目标页面召至前台。
- 二级管理流程仍可正常进入和返回。
- 自动测试与生产构建通过。
