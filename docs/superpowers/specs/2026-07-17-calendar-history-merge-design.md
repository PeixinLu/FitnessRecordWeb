# 日历与历史合并设计

## 目标

将独立的“历史”页面合并到“日历”页面。用户在日历中切换月份和选择日期后，可在同一页面下方查看该日期的训练数据。

## 页面与导航

- 底部导航移除“历史”，保留记录、日历、统计、设置四项。
- 移除 `/history` 路由；日历继续使用 `/calendar`。
- 日历默认选中当天，若切换月份时所选日期不在该月份，则选中该月第一天。

## 日历页布局

1. 顶部为月份切换控件和日期网格，训练日期继续显示标记。
2. 被选中的日期采用明确的选中态；今天保持独立标识。
3. 下方为“当日训练”区域，展示所选日期的训练记录。
4. 记录按动作聚类：每项显示动作名称、所属肌群、组数与数据摘要。
5. 所选日期无记录时，在该区域展示“当天暂无训练记录”空状态。

## 数据与交互

- 直接从现有 `recordStore.records` 按选中日期筛选并按 `exerciseId` 聚类，不新增数据库字段或迁移。
- 所有历史日期数据仅供查看，不提供删除、编辑、左滑操作或跳转至编辑页。
- 首页“今日训练”现有的编辑能力保持不变。

## 验证

- 增加静态测试，确认历史路由与 Tab 已移除，日历页包含受控选中日期及按动作聚类的当日训练区域。
- 运行全量 Node 测试与 `npm run build`。
