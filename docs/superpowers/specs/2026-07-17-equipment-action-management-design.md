# 器械动作管理设计

## 目标

新增一个器械动作管理页面，用户可管理器械及其包含的动作，并为每个动作选择训练数据模板。训练记录界面依据所选动作的模板显示并保存相应数据。

## 范围

- 设置页新增“器械动作管理”入口。
- 管理页支持新增、编辑、删除器械；删除器械时一并删除其动作。
- 管理页支持按器械查看、新增、编辑、删除动作。
- 每个动作保存所属肌群和一个内置数据模板。
- 内置模板固定为：
  - `weight-reps`：次数 + 重量
  - `reps`：仅次数
  - `duration`：时长（分钟）
  - `distance-duration`：距离（公里）+ 时长（分钟）
- 训练抽屉根据动作模板显示对应的录入与组详情字段。
- 训练记录持久化对应字段；既有重量和次数记录保持可读。

## 数据模型与迁移

- `Exercise` 增加必填字段 `dataTemplate`。旧的默认动作在数据库升级时补为 `weight-reps`。
- `WorkoutRecord` 新增可选 `duration` 与 `distance` 字段；继续保留 `reps` 和 `weight`，以兼容既有数据。
- Dexie 数据库升级到新版本，保留当前已有的器械、动作和记录数据。默认初始化的数据也都带有 `weight-reps`。

## 页面与交互

### 器械动作管理页

- 顶部为导航栏与“新增器械”按钮。
- 以单元格列表展示器械名称、动作数量；点击进入该器械的动作管理页。
- 器械支持编辑名称、删除。删除前使用确认对话框，并说明会删除其下动作。

### 动作管理页

- 展示所属器械，列出动作名称、肌群和模板名称。
- 支持新增、编辑、删除动作。
- 动作表单包含动作名称、肌群选择与数据模板选择，均为必填。
- 不允许保存空名称；删除动作前需确认。

### 训练抽屉

- 选择动作后，根据动作模板配置滚轮：
  - 次数 + 重量：次数、组数、kg。
  - 仅次数：次数、组数。
  - 时长：时长（分钟）、组数。
  - 距离 + 时长：距离（公里）、时长（分钟）、组数。
- 第二步的每组明细只显示该模板要求的字段，并以相同字段保存记录。

## 错误处理

- 表单字段缺失时提示用户并不写入数据库。
- 器械或动作删除后刷新 Pinia 中的器械与动作列表；若当前选择指向被删除项，则清空选择。
- 若某器械没有动作，训练抽屉仍保留现有的“请先选择动作”提示。

## 验证

- 为模板配置抽取纯函数并新增 Node 测试，覆盖四种模板的字段定义。
- 使用 `npm run build` 执行 TypeScript 检查与生产构建。
- 手动验证：器械/动作的增删改、模板选择、四种模板各录入一条记录，以及已有默认动作仍能记录重量训练。
