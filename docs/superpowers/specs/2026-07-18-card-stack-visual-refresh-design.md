# 卡片堆叠视觉刷新设计

## 目标

在不改变现有堆叠状态机、页面召回规则和二级路由的前提下，提升堆叠入口与四个一级页面的可识别性：

- 将右下角堆叠按钮改为浅色磨砂透明质感。
- 默认图标采用类似 Safari 多标签页的重叠卡片符号。
- 进入堆叠态后淡化为返回图标，并降低按钮透明度。
- 移除记录页顶部七天时间线。
- 为记录、日历、统计、设置统一增加左对齐大标题，使卡片在堆叠态仍可快速识别。

## 页面标题

新增一个仅负责一级页面标题的共享组件，四个页面分别传入以下名称：

- 记录
- 日历
- 统计
- 设置

标题位于每个页面自身内容的最上方，随卡片一同参与堆叠变换，而不是由堆叠容器悬浮覆盖。标题采用 34px 字号、41px 行高和 700 字重，左对齐显示；标题区域使用 18px 顶部、20px 左右和 10px 底部留白。

记录页移除原有七天日期时间线，以及只为该时间线服务的计算状态、触摸位置和滑动事件。大标题之后直接展示器械区域。其余页面在现有内容之前加入相同标题组件，页面内部功能和状态保持不变。

## 堆叠按钮

按钮保持右下角悬浮位置和现有可访问名称，不改变点击、键盘、焦点归位或召回锁定逻辑。

默认状态：

- `rgba(255, 255, 255, 0.82)` 半透明浅色背景。
- `blur(20px) saturate(180%)` 磨砂效果。
- 细浅色描边和柔和阴影。
- 深灰色重叠圆角矩形图标，表达多标签页或多卡片。

堆叠状态：

- 多标签页图标交叉淡出。
- 返回箭头交叉淡入。
- 按钮整体透明度降至 `0.82`，背景色保持浅色。
- 仍保留足够的浅色填充，避免黑色堆叠舞台大面积透过按钮。

图标使用组件内的 HTML/CSS 图形实现，不新增图片资源或第三方图标依赖。状态切换使用透明度和轻微缩放过渡；在 `prefers-reduced-motion` 下缩短过渡时间。

## 结构边界

- 共享标题组件只接受标题文本，不处理页面状态或导航。
- `PrimaryPageStack.vue` 只负责按钮视觉状态和已有堆叠交互。
- 四个页面继续拥有自己的内容、滚动和业务状态。
- 不改变 `cardStack.ts` 的堆叠状态模型。
- 不改变一级页面无历史切换和二级页面保留路由的现有规则。

## 自动化验证

本次不进行几何或浏览器验证，由用户完成最终视觉手动确认。自动化验证覆盖：

- 四个一级页面均使用统一标题组件和正确标题。
- 记录页不再包含七天时间线及其触摸处理。
- 堆叠按钮同时包含多标签页图标与返回图标，并由堆叠状态控制交叉淡化。
- 按钮使用浅色半透明、磨砂模糊、描边和阴影。
- 现有堆叠交互测试继续通过。
- `npm run build` 成功。
